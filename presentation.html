1B: Core Concept

What Is an Agent?

"An agent is an LLM that runs tools in a loop to achieve a goal."
— Simon Willison

Tools

It can do things — read files, run code, browse the web — not just generate text

A Loop

It tries something, sees the result, decides what to do next, and keeps going

A Goal

It's working toward an outcome, not just responding to your last message

Making It Click

The Restaurant Analogy

1

Chatbot = Vending Machine

Push button, get fixed item. No flexibility.

2

Assistant = Waiter

You tell them what you want, they relay it. Still following a script.

3

Agent = Personal Chef

You say "I'm in the mood for Italian." They check what's available, pick a recipe, cook it, taste it, adjust seasoning, and serve. If they're missing an ingredient, they substitute — without asking you.

The test: Can you give it a task and walk away? If yes, it's an agent.

1C: Key Framework

The Autonomy Slider

Products shouldn't offer a binary "AI on/off." They should offer a slider from full human control to full AI autonomy.

Like Tesla Autopilot — lane-keeping, then adaptive cruise, then lane changes. Users build trust gradually.

— Andrej Karpathy

The Autonomy Slider

Levels 1–3: Human Drives

1
Suggestion
AI predicts your next action, you accept or reject
Copilot autocomplete, spell check
2
Conversation
You describe what you want, AI generates it, you paste it in
ChatGPT, Claude chat
3
Integrated Partner
AI edits your work directly, you review each change
Cursor, Microsoft 365 Copilot
The Autonomy Slider

Levels 4–6: AI Drives

4
Supervised Agent
AI plans and executes multi-step work, asks permission at key moments
Claude Code ← we are here
5
Autonomous Agent
AI works independently, comes back with results
Devin, OpenAI Codex, background agents
6
Proactive Agent
AI notices things to do and acts without being asked
Scheduled agents, monitoring agents

The slider isn't about maximizing autonomy — it's about finding the right level for each task.

Why Each Step Matters

The Transitions

1→2

Suggestion → Conversation

"Instead of guessing what I'll type, I can just describe what I want." Non-technical people get access.

2→3

Conversation → Integrated Partner

"Instead of copy-pasting from a chat window, the AI edits my actual files."

3→4

Integrated Partner → Supervised Agent

"I describe the goal and it figures out the steps." The AI starts planning. This is the big jump.

4→5

Supervised → Autonomous

"I kick off a task and close my laptop. It keeps working."

5→6

Autonomous → Proactive

"I didn't ask it to do anything. It noticed something needed doing and did it."

Karpathy's Insight

We're NOT at full autonomy yet. And that's fine.

The most useful products right now are partial autonomy — you stay in the loop. The slider lets you ratchet up trust over time.

"Demo is works.any(), product is works.all()"

A demo only needs to work sometimes. A product must work reliably.

1D: Why Code First?

Why Does AI Start with Code?

1

Training Data

All of GitHub, Stack Overflow — the most abundant, machine-readable corpus ever.

2

Verifiability

Code runs or it doesn't. Tests pass or fail. Instant, objective feedback.

3

Fast Feedback Loops

Write → run → see error → fix, in seconds. Most domains take days or weeks.

4

Digital-Native

No physical world needed. Just read and write files.

5

Structured Output

Programming languages have grammars. The output space is constrained.

The Punchline

Most Knowledge Work Is Already Software

  • Spreadsheets are programming (VLOOKUP is a function, pivot tables are data pipelines)
  • Business workflows are software pipelines
  • Every knowledge worker's job is reading, thinking about, and producing digital files
Karpathy's Software 3.0

1.0: You write explicit code

2.0: You train neural networks on data

3.0: You describe what you want in English

If English is the programming language, the line between "coding" and "knowledge work" disappears.

Watch the Agent Pattern

$ cd ~/Desktop/demo && claude
claude> Create a beautiful HTML page that shows a dashboard with 3 charts showing fake quarterly sales data, with a dark theme
I'll create that dashboard for you...

What just happened in agent terms: It read the goal, planned the approach, wrote code, created a file — tools in a loop toward a goal. Same three properties from 1B.

The interaction: you describe, it does, you see the result, you iterate.

Hands-On

The Exercise

A non-coding business task done through Claude Code — proving the techniques transfer from coding to everything

Setup

The Dataset

$ cd ~/Desktop/tutorial
$ claude
Starting Claude Code...

What's in the CSV

~500 rows of simulated quarterly sales data: regions, products, sales reps, revenue, dates

Why This Task

It's a business task, not a coding task — directly proves the "techniques transfer" point

Phase 1: Exploration (~10 min)

Read & Summarize

claude> Read and summarize this dataset. What columns are there? How many rows? Any data quality issues?

Key observation: The agent reads the file, decides what analysis to run, writes and executes code to inspect the data, and reports back. You told it what you wanted to know, not how.

Phase 2: Analysis (~15 min)

Ask Business Questions

Try prompts like:

"Which region had the highest revenue growth quarter over quarter?"

"Who are the top 5 sales reps by total revenue?"

"Are there any seasonal patterns in the data?"

Watch for the loop: It writes Python code, runs it, sometimes hits an error, fixes the error, and tries again. This is the core agent behavior.

Phase 3: Visualization (~10 min)

Create Charts & Reports

claude> Create a chart showing revenue by region over time
claude> Make a presentation-ready summary with the top 3 insights

Then iterate: "Make the chart colors match our brand" / "Add a trend line"

Autonomy slider in action: You're at Level 4 (supervised agent). It's doing multi-step work, but you're reviewing and steering.

Phase 4: Go Wild (~15 min)

Go Wild

Your Own Data

Bring a spreadsheet or document from your own work

Creative Requests

"Write a memo summarizing these findings for my boss"

Push Boundaries

"What would you recommend we do based on this data?"

If the agent gets stuck in a loop: Great teaching moment! "This is why we're at Level 4, not Level 5."

Debrief

Quick Round-the-Table

  • What surprised you?
  • What worked better than expected? What didn't?
  • Did it feel like "programming"?

Connect back to Software 3.0 — you just "programmed" by describing what you wanted in English.
